2 · How we'll execute (two lanes)

The site runs on the classic Wix Editor, so the work splits into two lanes. The good news: the single highest-leverage AEO lever — publishing question-answering content with embedded schema — is fully in the lane I can execute directly.

LaneWhat it coversHow
🤖 APIBlog posts with per-post JSON-LD schema (FAQPage / Service), robots.txt, content-item SEO, Bookings service SEOI do it directly via the connected Wix MCP. No manual step.

5 · How we'll measure

The existing tracking harness re-runs the 40-query set across the AI engines and feeds the AEO Dashboard. Success = the visibility line climbs off ~7%, GV starts appearing on discovery (q1–q8) and problem-aware (q17–q24) queries, and new blog URLs show up as cited sources. We re-run the tracking after each content batch and watch:

  • Visibility score — % of non-branded queries where GV appears, per engine.
  • Citation share — % of organic citations pointing to gvskincare.com.
  • Query coverage — which query categories flip from absent → present.
Net: the work that moves AEO fastest — schema-rich, question-answering content — is exactly what I can publish directly through the Wix API. The static-page polish runs in parallel as paste-ready drafts. First measurable lift expected after the week-1 blog batch lands and the next tracking run completes.
